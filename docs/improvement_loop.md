# 改善ループ（AI駆動のブラッシュアップ機構）

AI（Claude Code）がカブログを継続的にブラッシュアップし、**方向性の判断は人間（あなた）が握る**ための機構。
本ドキュメントが**バックログの正本**であり、ループの**プロトコルの正本**。

設計思想は `docs/ai_driven_development.md` の延長。あの文書が「AIに実装させる側（インバウンド）」の
harness なら、本文書は「改善サイクルそのものを回す側（アウトバウンド）」の harness。
改善の駆動を規約でなく**構造（バックログ＋ゲート＋起動）**に落とす。

---

## 1. ループの形

```
①観測 → ②提案（候補+根拠+判断軸照合） → 〔③方向性ゲート＝あなた〕 → ④実行 → ⑤検証 → ⑥学習
   ↑                                                                              │
   └──────────────────────── 学習が次の観測を賢くする ──────────────────────────┘
```

各段の実体（大半は既存資産を使う。新規は②のバックログと③のゲートのみ）：

| 段 | すること | 使うもの |
|---|---|---|
| ①観測 | 改善の種を拾う（後述の3信号源） | `product_context.md` / `uiux_review.md` / `run-app` skill |
| ②提案 | 候補を並べ、根拠と判断軸照合を付ける | **本ファイルのバックログ表** |
| ③**ゲート** | あなたが go / 見送り / 方向修正 | **`/improve` の AskUserQuestion** |
| ④実行 | 実装する | Claude Code + agents + skills |
| ⑤検証 | AI が自分で確かめる | `pytest` / `verify` / `run-app` / `security-review` |
| ⑥学習 | 何が効いたか残す | 本ファイル §5 学習記録 / memory / doc 改訂ノート |

---

## 2. この機構の3つの決定（2026-07 確定）

方向性はあなたが決めた。以下は動かさない前提。変えるときは明示的に。

### 決定A：ゲートは「提案の前」に置く（AIの自律度）
**AI はコードを勝手に触らない。** ①観測→②提案（バックログへの候補追加）までを自走し、
あなたが候補に go/no-go を付けて**初めて④実行が走る**。
- 実装・migration・削除・リファクタは、バックログで `承認` になった項目のみ着手する。
- 未承認（`候補`）の項目を先走って実装してはならない。これが本ループの唯一の不変条件。

### 決定B：信号源は3つ（何を直すか、の起点）
①観測は次の3つだけを起点にする（コード健全性・技術的負債は本ループの起点から**外す**。
それらは `ai_driven_development.md §7 残課題` で別管理）。

1. **判断軸との差分** — `product_context.md` の問い
   「これは投資思考の**記録・蓄積・振り返り**に直接貢献するか？」に照らし、
   現状の画面・機能の穴（貢献できていない所、足せる所、引くべき所）を洗う。
2. **UI/UXレビュー** — `docs/uiux_review.md` の未対応項目＋初回体験（最初の5分）の穴。
3. **使用感（ドッグフーディング）** — あなたが実際に使って感じた摩擦、および
   `run-app` skill で AI が実機を触って見つけた引っかかり。

### 決定C：起動は「手動セッション起点」
**あなたが会話を始めた時だけ**ループを1周回す（`/improve` で起動）。
定期実行（cron/schedule）はしない。型が固まるまで手動で回し、過剰自動化を避ける。
自動化への移行は §5 学習記録が十分たまってから、別途あなたの判断で。

### 決定B-拡張（2026-07・R5）：コード健全性軸を追加
R1–R4 で「プロダクト機能をAI単独で探す」3信号源が doc-drift（実装済みの再発見）に
収束することが確定したため、ユーザー判断で **④コード健全性軸**（巨大ファイル分割・
死にコード除去・技術的負債）を起点に追加。この軸は「客観的・意図の誤読が起きない・
投機的でない」候補が出る唯一の領域で、AI駆動開発の速度・編集安全性に直結する。

## 4. `/improve` ラウンドの手順（プロトコル）

`/improve` を起動すると AI は1周を回す。詳細な起動手順は `.claude/commands/improve.md`。

1. **観測**：3信号源から候補を集める（判断軸チェック・`uiux_review.md`・必要なら `run-app`）。
   既存バックログの `候補` も棚卸しする。
2. **提案**：新候補を §3 バックログに `候補` で追記（判断軸照合を必ず付ける）。
3. **ゲート**：`AskUserQuestion` で、今回のラウンドで着手する候補をあなたに選ばせる
   （go / 見送り / 方向修正）。**ここで承認された項目だけ**が次に進む。
4. **実行**：承認項目を実装（既存規約・`AggregateService.deferred` 等の不変条件を守る）。
5. **検証**：`pytest` green・必要に応じ `run-app`/`verify`/`security-review`。バグ修正時はテスト追加（CLAUDE.md 規約）。
6. **学習**：結果と学びを §5 に1行記録。バックログの状態を更新。
